◆ Callback_Dsp_Settings_t

typedef void(* Callback_Dsp_Settings_t) (void *context, int32_t protocol_handle, uint8_t endpoint, const DSP_Settings_t *dsp_settings_ptr)

This functions registers a callback for algorithm DSP settings defined by DSP_Settings_t.

If a callback for this message type is registered, that callback is issued every time, a connected board requests current DSP settings from device using .If no callback is registered the message is ignored.

Whenever a callback is issued, the following parameters are passed to the callback:

Parameters
[in]contextThe context data pointer, provided along with the callback itself through ep_targetdetect_set_callback_dsp_settings.
[in]protocol_handleThe handle of the connection, the sending device is connected to.
[in]endpointThe number of the endpoint that has sent the message.
[in]dsp_settings_ptrCurrent DSP settings used by algorithm on target is returned into this structure.

This function gets the DSP settings for the algorithm used in the device.

DSP settings is defined by struct DSP_Settings_t. This DSP structure is returned via callback which user has register defined by Callback_Dsp_Settings_t.

Parameters
[in]protocol_handleA handle to the connection to the sensor device.
[in]endpointThe endpoint in the connected device to be checked for compatibility.
Returns
If the specified endpoint is compatible to this implementation the function returns 0. If the endpoint is not compatible, a negative error code is returned.
