◆ radar_set_chirp_time()

uint16_t radar_set_chirp_time ( Radar_Handle_t  device,
const uint32_t  chirp_time_usec 
)

This function sets the chirp-time, used by PLL for linear FMCW chirp and CW Doppler chirp.

Changing chirp-time also changes the sampling frequency of the FMCW. There are some limits on this chirp time because of MCU processing capabilities and PLL configuration limitations.

Parameters
[in]deviceA handle to the radar sensor device.
[in]chirp_time_usecAn unsigned 32-bit value, in units of microseconds (usec) which should be within the limits of BSP_MIN_CHIRP_TIME_USEC & BSP_MAX_CHIRP_TIME_USEC.
Returns
One of the following status can be return:
  • RADAR_STATUS_SUCCESS Success
  • RADAR_STATUS_FAIL Fail
  • RADAR_STATUS_NA Feature not available

This function starts the acquisition of data.

This function sets the Frame Interval period . Afterwards starts the timer for data acquisition trigger.A valid frame interval must be provided by the user in config.h file.

Parameters

Returns

  • RADAR_STATUS_SUCCESS Success

◆ radar_stop_acquisition()

uint16_t radar_stop_acquisition ( Radar_Handle_t  device)

This function stops the acquisition of data.

This function sets the Frame Interval period to Zero and stops the timer for data acquisition trigger through the firmware base API method radar_set_automatic_frame_trigger().

Parameters
[in]deviceA handle to the radar sensor device.
Returns
An error code indicating if the function succeeded. The following error codes can occur:
  • RADAR_ERR_OK if the a frame of radar data could be returned
  • RADAR_ERR_UNSUPPORTED_FRAME_INTERVAL if the internal timer can't be configured to the given frame interval
  • RADAR_ERR_BUSY if the device is not ready for data acquisition (maybe it's in test mode)
