◆ Chirp_Direction_t

enum Chirp_Direction_t

This enumeration type lists all possible chirp directions in FMCW radar mode.

A value of this type is part of the struct Fmcw_Configuration_t.

To use the modes RADAR_DIR_ALTERNATING_FIRST_UP and RADAR_DIR_ALTERNATING_FIRST_DOWN, the number of chirp per frame should be at least 2, otherwise these modes behave the same way as RADAR_DIR_UPCHIRP_ONLY and RADAR_DIR_DOWNCHIRP_ONLY.

Enumerator
RADAR_DIR_UPCHIRP_ONLY 

All chirps of a frame start at the lower FMCW frequency and ramp up to the upper FMCW frequency.

RADAR_DIR_DOWNCHIRP_ONLY 

All chirps of a frame start at the upper FMCW frequency and ramp down to the lower FMCW frequency.

RADAR_DIR_ALTERNATING_FIRST_UP 

Only the odd chirps of a frame start at the lower RF frequency and ramp up to the upper FMCW frequency. The even chirps of a frame go from the upper to the lower FMCW frequency.

RADAR_DIR_ALTERNATING_FIRST_DOWN 

Only the even chirps of a frame start at the lower RF frequency and ramp up to the upper FMCW frequency. The odd chirps of a frame go from the upper to the lower RF frequency.

This function returns the FMCW ramp speed in in current configuration.

This function writes the RF frequency change per second to an integer variable. The variable must be provided by the user and may be uninitialized.

The unit of chirp duration value is MHz/s.

While automatic trigger is active, reading the chirp duration may fail.

Parameters
[in]deviceA handle to the radar sensor device.
[out]bandwidth_per_second_MHz_sA pointer to a location, where the FMCW ramp speed value will be stored.

◆ radar_set_fmcw_configuration()

uint16_t radar_set_fmcw_configuration ( Radar_Handle_t  device,
const Fmcw_Configuration_t *  configuration 
)

This function modifies the RF parameters in FMCW radar mode.

The function changes the RF parameters for FMCW radar mode according to the provided structure of type Fmcw_Configuration_t. After calling this function, the device will be in FMCW radar mode.

A value check is applied to each field of configuration. If any value or the combination of the values is out of range, the FMCW parameters are not changed and an error is returned.

The configuration can not be changed while the automatic trigger is active.

Parameters
[in]deviceA handle to the radar sensor device.
[in]configurationA pointer to the new FMCW parameters to be applied.
Returns
An error code indicating if the function succeeded. The following error codes can occur:
  • RADAR_ERR_OK if the FMCW parameters could be applied
  • RADAR_ERR_BUSY if the automatic trigger is active and the device is busy with acquiring radar data
  • RADAR_ERR_FREQUENCY_OUT_OF_RANGE if the upper and/or lower FMCW frequency is not in the radar sensor's RF range
  • RADAR_ERR_POWER_OUT_OF_RANGE if the specified TX power value exceeds the maximum value
  • RADAR_ERR_UNSUPPORTED_DIRECTION if the device does not support the requested chirp direction
