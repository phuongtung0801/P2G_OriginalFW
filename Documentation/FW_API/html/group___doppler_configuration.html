This function returns the current RF parameters in Doppler radar mode.

This function returns the current RF parameters in Doppler mode through a struct of type Doppler_Configuration_t. The struct must be provided by the user and may be uninitialized.

While automatic trigger is active, reading the current frame format may fail.

Parameters
[in]deviceA handle to the radar sensor device.
[out]configurationA pointer to a location, where the current Doppler configuration will be stored.

const Doppler_Configuration_t *  configuration 
)

This function modifies the RF parameters in Doppler radar mode.

The function changes the RF parameters for Doppler radar mode according to the provided structure of type Doppler_Configuration_t. After calling this function, the device will be in Doppler radar mode.

A value check is applied to each field of pConfiguration. If any value or the combination of the values is out of range, the FMCW parameters are not changed and an error is returned.

The configuration can not be changed while the automatic trigger is active.

Parameters
[in]deviceA handle to the radar sensor device.
[in]configurationA pointer to the new Doppler parameters to be applied.
Returns
An error code indicating if the function succeeded. The following error codes can occur:
  • RADAR_ERR_OK if the FMCW parameters could be applied
  • RADAR_ERR_BUSY if the automatic trigger is active and the device is busy with acquiring radar data
  • RADAR_ERR_FREQUENCY_OUT_OF_RANGE if the upper and/or lower FMCW frequency is not in the radar sensor's RF range
  • RADAR_ERR_POWER_OUT_OF_RANGE if the specified TX power value exceeds the maximum value
